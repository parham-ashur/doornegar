| # | Finding | Location | Action |

## 4. Data pipeline integrity

### Verified state
- Temperature for bias scoring was **0.3** (not deterministic) — confirmed at `bias_scoring.py:397`. Fixed to `0` in the same patch as this audit.
- Ingest loop at `ingestion.py:198` had no per-source try/except — one exception would abort the remaining sources in a run. Wrapped in the same patch.
- Celery beat schedule runs ingest every 15 min, NLP every 15 min, clustering every 30 min, scoring every 60 min (`celery_app.py:24-53`). **No Redis lock on any task** — concurrent runs are possible if one slows.
- `Story.telegram_analysis` is a write-once JSONB cache with **no invalidation path anywhere** in the codebase.

### Blockers
| # | Finding | Location | Status |
|---|---|---|---|
| P1 | Bias scoring `temperature=0.3` broke determinism claim made in prompt | `backend/app/services/bias_scoring.py:397` | **Fixed this session** → `0` |
| P2 | Ingest loop: one crashing source aborted the entire run | `backend/app/services/ingestion.py:198` | **Fixed this session** → per-source try/except |
| P3 | Telegram analysis cache is permanent write-once; new posts never refresh it | `backend/app/api/v1/social.py` | **Fixed this session**: `cached_at` stamp written into the JSONB, 48h TTL on read, `force_refresh=1` admin bypass, new `POST /social/stories/{id}/telegram-analysis/invalidate` admin route |

### Risks
| # | Finding | Location | Action |
|---|---|---|---|
| P4 | Celery beat schedule has no lock — if ingest exceeds 15 min, second run starts mid-commit | `backend/app/workers/celery_app.py:24-53` | **Fixed this session**: new `workers/task_lock.py` helper; `@single_flight` decorator on every beat task with timeout slightly above the task's hard time_limit |
| P5 | Articles stuck with `llm_failed_at` > N days have no dashboard surface — pipeline drift is invisible | `backend/app/models/article.py:51`, admin dashboard | Add "stuck failures >24h" widget |
| P6 | Single unmatched articles (`story_id IS NULL`) are never reassessed after the first clustering attempt | `backend/app/services/clustering.py:720-732` | **Partially fixed this session**: clustering stats now include `aged_orphans` count + WARN log. Full fix (synthetic "miscellaneous" story) deferred — surface is now visible |
| P7 | Dispute score is still the `|state_pct − diaspora_pct|` proxy (already tracked in memory) — LLM-computed version pending | `backend/app/services/clustering.py` | Matches existing project_dispute_score memory — prioritize in next sprint |
| P8 | Deduplication is URL-only; republishes under new URL show as separate articles | `backend/app/models/article.py:26` | Secondary dedup on title+source hash after URL conflict miss |

### Nice-to-have
- `clustering_similarity_threshold=0.45` config and a hardcoded `0.30` pre-filter in `clustering.py:330` — two thresholds in two places. Consolidate.
- Stories have no archive / soft-delete — 30-day-stale stories stay ranked.
- Blindspot threshold (`state_pct < 10`) is a magic number — parameterize + doc.
